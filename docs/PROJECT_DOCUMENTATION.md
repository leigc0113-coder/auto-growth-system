## 1. 项目概述

### 1.1 项目目标
**Auto Growth System** 是一个全自动化的用户增长系统，专为 Telegram 平台设计，实现：
- ✅ 自动采集目标用户/频道
- ✅ AI生成个性化推广内容
- ✅ 多渠道自动化推广
- ✅ 智能防封策略
- ✅ 数据追踪与分析

### 1.2 核心优势
| 优势 | 说明 |
|------|------|
| **全自动** | 无需人工干预，24/7自动运行 |
| **多账号** | 支持多个账号轮换，降低封号风险 |
| **AI驱动** | 智能生成推广文案，提高转化率 |
| **防封策略** | 智能限速、冷却机制、异常检测 |
| **数据追踪** | 完整转化漏斗，ROI可量化 |

## 2. 系统架构

### 2.1 六层架构设计

```
┌─────────────────────────────────────────────────────────┐
│  Phase 1: 数据层 (Data Layer)                           │

│  ├─ 私信推广             → 主动私信目标用户            │
│  ├─ 群组互动             → 加群+自然发言               │
│  ├─ 帖子评论             → 在相关帖子下引流            │
│  └─ 频道邀请             → 邀请用户加入频道            │

│  Phase 4: 管理层 (Management Layer)                     │

│  └─ 用户标签             → 分级+优先级评分             │
├─────────────────────────────────────────────────────────┤
│  Phase 5: 调度层 (Scheduler Layer)                      │
│  ├─ 定时采集             → 每日02:00自动执行           │

├─────────────────────────────────────────────────────────┤
│  Phase 6: 分析层 (Analytics Layer)                      │
│  ├─ 转化追踪             → 完整转化漏斗                │
│  └─ 数据分析             → 统计+趋势+ROI               │
└─────────────────────────────────────────────────────────┘
```

## 3. 核心功能详解

### 3.1 Phase 1: 数据层

#### 3.1.1 Stealth采集器
**功能**: 使用 Puppeteer Stealth + 住宅代理绕过 Cloudflare 反爬

**采集平台**:
- Telemetr.io
- TGStat

**采集数据**:
- 频道名称
- 用户名 (@username)
- 订阅数
- 频道描述
- 关键词标签

**技术亮点**:
- ✅ 绕过 Cloudflare 验证
- ✅ IPRoyal 住宅代理
- ✅ 真人行为模拟（随机滚动）
- ✅ 已采集 321 个频道

#### 3.1.2 数据存储
- **数据库**: MongoDB
- **存储内容**: 目标列表、推广记录、转化数据
- **去重机制**: 基于 username 自动去重

---

### 3.2 Phase 2: 内容层

#### 3.2.1 AI内容生成器
**技术**: Kimi API (Moonshot)

**支持内容类型**:
| 类型 | 用途 | 语言支持 |
|------|------|---------|
| 私信推广 | 给管理员发合作邀请 | 英文/印地语 |
| 邀请消息 | 邀请加入频道 | 英文/印地语 |
| 评论内容 | 在帖子下自然引流 | 英文/印地语 |
| 自动回复 | 回答用户咨询 | 英文/印地语 |

**备用机制**: 当 Kimi API 不可用时，自动切换到预设模板（12套英文+12套印地语）

#### 3.2.2 自动回复系统
- **关键词匹配**: 预设常见问题自动回复
- **AI对话**: 智能回答用户问题
- **上下文管理**: 维护对话历史

---

### 3.3 Phase 3: 推广层

#### 3.3.1 私信推广 (Direct Message)
**目标**: 频道/群组管理员
**内容**: 合作邀请、交叉推广
**审核机制**: 发送前需管理员确认

#### 3.3.2 群组互动 (Group Engagement)
1. 主动加入目标群组
2. 发送自然互动消息
3. 建立信任后引流

#### 3.3.3 帖子评论 (Comment)
- 在相关帖子下评论
- 自然提及自己的频道
- 不直接广告，避免被删

#### 3.3.4 频道邀请 (Channel Invite)
- 生成一次性邀请链接
- 邀请用户加入频道
- 追踪邀请转化率

---

### 3.4 Phase 4: 管理层

#### 3.4.1 多账号管理
**当前配置**:
| 账号 | 手机号 | 角色 | 功能 |
|------|--------|------|------|
| 账号1 | +919799524106 | 私信专用 | 发送私信推广 |
| 账号2 | +917416201930 | 加群专用 | 加入群组+发言 |

**轮换机制**: 根据功能自动分配任务到最合适的账号

#### 3.4.2 反封策略
| 功能 | 冷却期 | 每日限额 |
|------|--------|---------|
| 私信 | 60秒 | 20条 |
| 加群 | 5分钟 | 5个 |
| 发言 | 2分钟 | 30条 |
| 邀请 | 3分钟 | 10个 |

**异常处理**:
- 连续5次错误自动暂停1小时
- 错误率超过30%自动暂停
- 支持手动恢复

#### 3.4.3 用户标签
- **分级**: mega(10万+), large(1万+), medium(1千+), small(100+), micro
- **分类**: gaming, money, crypto, indian, lottery
- **优先级评分**: 0-100分

### 3.5 Phase 5: 调度层

**定时任务 (IST时区)**:
| 时间 | 任务 | 说明 |
|------|------|------|
| 02:00 | 自动采集 | 采集新频道，存入数据库 |
| 10:00 | 自动推广 | 生成任务，发送审核通知 |
| 18:00 | 数据报告 | 发送日报到管理员 |

**通知方式**: Telegram私信

### 3.6 Phase 6: 分析层

#### 3.6.1 转化追踪
**转化漏斗**:
```
曝光 → 点击 → 互动 → 加入 → 活跃
```

**追踪指标**:

**日报内容**:

## 5. 工作流程

### 5.1 每日全自动流程

```
02:00 ┌───────────────────────────┐
      │  定时采集任务启动          │
      │  - Stealth采集Telemetr    │
      │  - 解析频道信息            │
      │  - 存入数据库              │
      └───────────┬───────────────┘
                  ↓
10:00 ┌───────────────────────────┐
      │  定时推广任务启动          │
      │  - 读取未推广目标          │
      │  - AI生成推广文案          │
      │  - 添加到审核队列          │
      └───────────┬───────────────┘
                  ↓
      ┌───────────────────────────┐
      │  发送Telegram通知给你      │
      │  "新推广任务待审核"         │
      └───────────┬───────────────┘
                  ↓
      ┌───────────────────────────┐
      │  你回复 /approve 批准      │
      └───────────┬───────────────┘
                  ↓
      ┌───────────────────────────┐
      │  系统执行推广              │
      │  - 账号1: 发送私信         │
      │  - 账号2: 加入群组+发言    │
      └───────────┬───────────────┘
                  ↓
      ┌───────────────────────────┐
      │  记录结果到数据库          │
      └───────────┬───────────────┘
                  ↓
18:00 ┌───────────────────────────┐
      │  生成数据报告              │
      │  发送日报给你              │
      └───────────────────────────┘
```

### 5.2 人工审核流程

```
系统自动生成任务
       ↓
发送Telegram通知:
"🔔 新推广任务待审核

类型: directMessage
目标: @SomeChannel
内容预览: Hi! ...

请回复:
✅ /approve abc123 - 批准
❌ /reject abc123 - 拒绝"
       ↓
你回复 /approve abc123
       ↓
系统自动发送
       ↓
发送结果通知
```
